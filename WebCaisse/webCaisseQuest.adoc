= Question supplementaire

= Cas WebCaisse : Java SQL et securité

== NF525
La NF525 dit que le logiciel caisse doit garder toutes ventes et changements.  
Pour WebCaisse ca veut dire quon peut pas effacer ou modifier une vente sans trace sinon cest pas conforme.  
Donc tables historiques et journaux pour savoir qui a fait quoi et quand.

== Mot de passe par mail
Envoyer mot de passe par mail clair ce'st une faille grave  
Quelqun peut voir ou pirater le mail  
Cest un probleme de confidentialite et dintegrite car autre peut se connecter et modifier données  
Et RGPD dit que les mot de passe clair interdit  
Donc une meilleur solution: client rempli formulaire, systeme hash mot de passe, envoie lien unique activation temporaire, client clique et choisit mot de passe, hashé stocké  
Lien périme apres usage donc sécurisé

== Table FormuleSouscrite
Avant table c juste idPointDeVente et idFormule  
Probleme: pas de dateDebut ni dateFin, donc on sait pas quand formule active ni combien de temps  
Facturation mensuelle marche pas

== Correction table
FormuleSouscrite(idSouscription, idPointDeVente, idFormule, dateDebut, dateFin)  
idSouscription cle primaire, idPointDeVente et idFormule cle etrangere  
On peut garder historique et calculer prix selon temps

== Trigger
Trigger audit apres update sur FormuleSouscrite  
Insert dans AuditChangementFormule idPointDeVente, ancienneFormule, nouvelleFormule, dateChangement  
Permet traçabilité et conforme NF525

== WHERE vs HAVING
WHERE filtre avant groupement, HAVING filtre apres groupement  
Dans listeConsoAFideliser on met HAVING COUNT>seuil car on compte ventes par conso

== Concatenation SQL
Si on fait "... having count(*) > " + seuil, utilisateur peut mettre code malicieux  
Injection SQL possible (OWASP)  
Donc PreparedStatement obligatoire

== PreparedStatement
Paramètres séparés du SQL, injection impossible  
Plus rapide et fiable que Statement classique

== instanceof
instanceof permet savoir type objet  
Dans compareLieuVente on sait si VenteEcommerce ou VenteMagasin  
Aide pour calcul, mais pas super propre, mieux méthode polymorphe dans Vente

== Division par zero
Si totalEcom = 0 alors totalMag/totalEcom plante  
Faut verifier avant sinon exception ou renvoyer 0

== Launcher et Main
Launcher demarre JavaFX correctement  
Main contient start() pour scene et controllers  
Sinon JavaFX crash

== Test unitaire
testInitConso verifie pointsFidelite=0 à creation ConsoFidele  
testAddMontant verifie points selon type programme  
1 tampon, 2 montant, 3 points  
Exemple: 10 pts pour 100-200€, 20 pts 201-500€, 50 pts au dessus

== SQL exemples
Liste conso 2017: SELECT DISTINCT c.nom,c.prenom,c.mail FROM Conso c JOIN Vente v ON v.idConso=c.id WHERE YEAR(v.dateVente)=2017  
Nb conso fidel 18-30: SELECT COUNT(*) FROM ConsoFidele WHERE TIMESTAMPDIFF(YEAR,dateNaiss,CURDATE()) BETWEEN 18 AND 30
