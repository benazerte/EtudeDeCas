= TP Étude de Cas : Coopain

== Mission 1 - Gestion des demandes d’insémination

=== Question 1.1

La requête montre comment Coopain choisi une paillette.

En gros ils prennent :
- une paillette du taureau demandé
- du type de paillette demandé
- avec au moins 1 en stock

Et ensuite ils regardent la date du lot et ils prennent **le plus ancien** (date minimale).

Donc la règle de gestion c’est :  
pour une demande d’insémination, on prend une paillette dispo du bon taureau et du bon type, et on utilise d’abord le lot le plus ancien encore en stock.

SELECT tp.libelle, t.prixPaillette
FROM Tarif t
JOIN TypePaillette tp ON t.idTypePaillette = tp.id

----

=== Question 1.4

Requête qui donne la liste des taureaux avec le stock total restant, trié du plus petit au plus grand.

[source,sql]
----
SELECT ta.idNational, ta.nom, SUM(l.stockRestant) AS stockTotalRestant
FROM Taureau ta
JOIN Lot l ON ta.idNational = l.idNationalTaureau
GROUP BY ta.idNational, ta.nom
ORDER BY stockTotalRestant ASC;
----

=== Question 2.3

Si le téléphone est perdu ou volé, ça peut poser des problèmes avec la base SQLite.

Confidentialité :

Une personne peut accéder à :
- infos sur les tournées
- identités des inséminateurs
- véhicules
- activité de Coopain

Intégrité :

Quelqu’un pourrait :
- modifier les km
- supprimer des tournées
- ajouter de fausses données
- envoyer de mauvaises infos au serveur

Solutions possibles :

- chiffrement du téléphone
- code PIN ou empreinte
- gestion des mobiles (MDM)
- effacement à distance
- stocker le moins de données possible en local

=== Question 2.4

Stocker un mot de passe en clair c’est très dangereux.

Pourquoi :

- si la base est volée → les mots de passe sont lisibles
- si quelqu’un intercepte le réseau → il peut voir le mot de passe
- les gens réutilisent souvent leurs mots de passe

Donc ça peut compromettre plusieurs comptes.

Solutions :

- jamais stocker le mot de passe en clair
- stocker seulement un hash
- utiliser HTTPS / TLS
- authentification sécurisée côté serveur

public ArrayList<Adherent> getAdherents() {

    ArrayList<Adherent> lesAdherents = new ArrayList<>();

    for (Visite v : laTournee.getLesVisites()) {

        Adherent a = v.getLeAdherent();

        if (!lesAdherents.contains(a)) {
            lesAdherents.add(a);

=== Question 3.3

Une attaque possible est une **attaque Man-in-the-Middle**.

Un attaquant placé entre le smartphone et le serveur peut :
- intercepter les données
- lire les infos
- modifier les données avant qu’elles arrivent au serveur

Pour éviter ça, il faut sécuriser les échanges avec **HTTPS et TLS**.
